### Work28

Домашка по CodeFirst:

3хуровневое приложение.

всего 4 проекта:

1) Console Apllication (В нем Main и в него подключается PL)
2) Presentation Layer (GroceryStorePL)
3) Business Logic Layer (GroceryStoreBL)
4) DAL (GroceryStoreDAL)

Существует магазин драгоценных изделий.
Изделие (Product): имеет Название, цену, тип (кольцо, серьга, ожерелье и т.д.) а так же в нем может быть от 0 до 10 драгоценных камней.
Драгоценный камень: имеет размер, цену, цвет.
Тип изделия - только название.

Реализовать:
Получение продукта по Id
Получение продуктов по типу
Получение продуктов по наличию/отсутвию в нем драгоценных камней.

т.е. В PL должно быть всего 3 этих метода.

Пользователь должен увидить Полную стоимость продукта (цена товара + сумма стоимости всех камней). 
Отдельно цену драгоценных камней пользователю знать нельзя.

Должно быть 3 миграции:
1) InitialCreate (на данной миграции создается только Product с полями: Price(decimal), Name(string) и Type(string))
2) MoveTypeToTable(в этой миграции перенос Type в отдельную таблицу). (эта миграция должна проходить и Up и Down без потери данных)
3) AddGemstones(создание таблицы GemStone и всех её связей)

Чтобы было проще:
ProductViewModel {
public int Id { get; set; }
public string TypeName { get; set; }
public decimal Price { get; set; }
}

Использовать жадную загрузку.
